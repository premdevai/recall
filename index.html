How it works

One extraction. One dataset. Every format.

Recall reads your history into a single evidence model, then renders it. Every source is just an adapter that maps onto that model — add one without touching the renderers.

sources ────────────► evidence.json ────────────► renders git (always) │ ├── journey.html (self-contained page) GitHub / GitLab / Bitbucket │ one schema: └── journey.md (paste-ready text) Jira / Linear │ accomplishments + Datadog / observability │ evidence[] + CI / CD │ metrics{} + skills

Install

Two commands, zero config.

No API keys, nothing to build. Recall runs as a Claude Code skill and reuses the git and MCP connections you already have.

Recommended

npx

Installs the skill in one command, then ask Claude /recall.

# installs to ~/.claude/skills
npx @premdevai/recall

# then, in any repo
/recall

Why it holds up

Proof, not a self-rating.

Evidence or it didn't happen

Every claim links back to a commit, PR, ticket, or metric. Skill depth is real merged-PR and commit counts — never a made-up 0–100 score.

Confidence, scored

Each contribution is ranked on how résumé-worthy it is. A formatting change lands near 2; a shipped feature with measured impact near 95. The trivia stays off the page.

Private by design

Runs entirely locally. Reads only what's on your machine or in the tools you've connected, and sends nothing anywhere except the model call you initiate. No servers, no telemetry.

Walk in with the receipts.

Clarity, confidence, and proof behind every line — reconstructed from the work you've already done.
